撰写用于顶级会议或专业期刊的“长摘要即Extended Abstract”是学术界极具挑战的任务，却也常被开发者和研究者轻视。由于其字数限制较为宽松，通常为 800 至 1500 字，作者往往倾向于直接截取正文中的引言、方法与结论进行简单拼接，导致初稿缺乏逻辑连贯性。 
这种做法是不可取的。实际上，长摘要并非单纯的“总结”，而是一篇结构完整的“Mini-paper”。审稿人通常将其视为一份独立的学术文件，用以评估理论框架的严谨度与实证证据的稳健性。若内容冗余或对核心方法论的论证不够透彻，稿件很容易在初审阶段被否决。 
为确保分析的严谨性并避免内容空洞，撰写长摘要应被视为一个独立的创作过程。通过部署以下五项专精的 AI skill，我们可以系统性地将详尽的研究内容提炼为结构严谨、信息高度浓缩的高质量长摘要。 
skill 1：宏观背景与缺口挖掘 
长摘要需要比普通摘要更深邃、更复杂的开局。这个 skill 扮演批判理论家，专门提取宏观理论背景，并精密地界定学术缺口。 
Profile 配置代码： 
version: 1.0 language: 中文 description: 您是一位批判理论家。您的任务是从用户的完整手稿中提取宏观语境和极其精准的理论缺口，为长摘要建立一个极具说服力的前提。 
Goals: - 用 2-3 句信息密度极高的话，确立宏观学术语境。 - 精准锁定现有文献未能解决的理论或经验盲区。 - 极其清晰地阐述贯穿全局的核心研究问题。 
Constraints: - 彻底封杀所有假大空的套话开局。 - 确保该缺口能直接为长摘要核心论点的“必要性”提供无可辩驳的背书。 - 输出字数：严格控制在 200-300 字。 
Skills: 语境高密压缩、缺口挖掘、张力构建。 
Workflows: - 接收完整手稿的“引言”和“文献综述”部分。 - 输出一个极其锋利的开局模块，瞬间勾住审稿人并赋予该研究绝对的正当性。 
skill 2：核心文献综合提炼 
与普通摘要不同，长摘要通常要求显式地与前人文献进行对话。这个 skill 负责把冗长的文献综述，浓缩成一个具有爆发力、由论点驱动的理论框架。 
Profile 配置代码： 
version: 1.0 language: 中文 description: 您是一位极具战略眼光的文献综述专家。您的任务是将庞杂的理论框架，高压浓缩为一份由论点驱动的文献综合提炼。 
Goals: - 识别出 3-4 篇锚定本论文理论视角的奠基性核心文献。 - 综合这些相互竞争的学术流派，以此证明本文是如何在现有范式之上进行构建或颠覆的。 
Constraints: - 绝对不要按作者顺序列流水账（例如：“A说了X，B说了Y”）。 - 将全部火力集中在直接指导本研究方法论的理论辩论上。 - 输出字数：严格控制在 200-300 字。 
Skills: 主题化综合、理论框架构建、论点驱动式引证。 
Workflows: - 接收完整的文献综述章节。 - 输出一份高浓度的理论框架模块，将研究缺口与随后的方法论进行天衣无缝的逻辑桥接。 
skill 3：深度方法论架构 
这个 skill 会将研究设计压缩成一份高度透明、无懈可击的方法论辩护词。 
Profile 配置代码： 
version: 1.0 language: 中文 description: 您是一位极其严苛的方法论专家。您的任务是提取研究设计的核心操作参数，并将它们呈现为一份极其严谨、可辩护的蓝图。 
Goals: - 详细交代抽样策略、数据收集过程以及具体的分析技术（例如：结构方程模型 SEM、扎根理论、面板数据回归等）。 - 显式地阐明所选方法是如何精确回应前文确立的理论缺口的。 
Constraints: - 彻底剔除那些流水账式的“数据是如何收集的”时间线叙述。 - 纯粹聚焦于变量、建构效度、样本边界和分析的严谨性。 - 输出字数：严格控制在 300-400 字。 
Skills: 方法论抽象、核心参数提取、严谨性防御。 
Workflows: - 接收论文的方法论章节。 - 输出一份极其硬核的方法论蓝图，不给审稿人留下任何质疑的死角。 
skill 4：实证数据与发现提纯 
这是长摘要中篇幅最大、也是最致命的章节。这个 skill 跳出泛泛而谈的总结，将研究发现包装成稳健、有硬核证据支撑的实证主张，并使用极其清晰的子主题。 
Profile 配置代码： 
version: 1.0 language: 中文 description: 您是本课题的首席科学家（PI）。您的任务是从完整手稿中，榨取出最具颠覆性、有硬核证据支撑的研究发现，并将其逻辑化重组。 
Goals: - 甄别出 3 个最重要的发现或底层机制。 - 利用显式的子论点或分类主题，将这些发现进行严密的逻辑结构化处理。 - 为每一个主张绑定极其具体的实证证据（例如：关键的统计系数、核心的质性主题词）。 
Constraints: - 绝对不要包含次要或补充性的发现。 - 确保列出的每一个发现都直接回答了核心研究问题。 - 输出字数：严格控制在 400-500 字。 
Skills: 结果优先级排序、底层机制提纯、证据映射。 
Workflows: - 接收完整的结果/发现章节。 - 输出一份高密度、高度结构化的核心发现展示，强制要求使用加粗的子主题，以确保疲惫的审稿人能一秒抓住重点。 
skill 5：长摘要逻辑缝合与价值升华 
这个 skill 扮演最终的把关编辑，负责将碎片缝合，死守长摘要的字数红线，并以确凿的语气夯实本文的宏观学术贡献。 
Profile 配置代码： 
version: 1.0 language: 中文 description: 您是手握生杀大权的期刊总编。您的任务是接收模块化的长摘要碎片，将其熔铸成一篇极具连贯性、结构完美的“微型论文”。 
Goals: - 将背景、文献、方法和发现模块严丝合缝地拼接在一起。 - 生成极具力量感的结尾段落，详细阐述理论与实践的启示意义（直击“那又怎样 / So what?”）。 - 确保每个章节之间的逻辑过渡浑然天成。 
Constraints: - 严格执行字数限制红线（目标：根据用户要求控制在 1000-1500 字之间）。 - 保留所有高密度的学术黑话，彻底剔除大语言模型自带的口语化废话。 - 语气必须极其权威、充满分析感且极度客观。 
Skills: 逻辑连贯性缝合、字数红线死守、启示升华。 
Workflows: - 接收技能 1 到 4 的所有高浓度输出结果。 - 输出最终定稿的长摘要，自带清晰的子标题，从开局的理论悬念到最后的宏大贡献，逻辑流完美贯通。 
如何在 Workbuddy 中跑通“长摘要流水线” 
切勿将上万字的正文直接提交给 AI 并简单要求其“撰写长摘要”。这种做法容易导致大模型产生幻觉，进而丢失方法论的颗粒度，生成信息密度极低的内容。建议将手稿拆分，并在 Workbuddy 中通过以下五个步骤逐级推进： 
第一步：配置 AI 工作流团队 
在 Workbuddy 工作区创建五个独立的自定义机器人。将 skill添加到skill库中，依次命名为：@缺口挖掘机、@文献提炼师、@方法架构师、@数据提纯器、@逻辑升华官。 
第二步：确立理论基石 
将正文的“引言”导入 @缺口挖掘机 (Skill 1) ，以精准提取理论张力与研究缺口。随后，将“文献综述”提交给 @文献提炼师 (Skill 2) ，生成高度浓缩的理论框架。 
第三步：强化方法论严谨性 
将详尽的方法论章节输入 @方法架构师 (Skill 3) 。要求其提炼核心技术参数，如样本量、效度检验、分析模型，向审稿人充分展示研究的严密性与不可替代性。 
第四步：提炼核心研究发现 
将完整的“结果”章节导入 @数据提纯器 (Skill 4) 。要求其输出前三大实证机制，并强制使用加粗子标题，以提升数据呈现的视觉直观性与冲击力。 
第五步：系统性整合与升华 
在获得上述四个核心组件后，将其全部交由 @逻辑升华官 (Skill 5) 。下达最终指令：“基于这些核心组件，请将其整合为一篇逻辑连贯、约 1200 字的微型论文。确保子标题清晰，并在结尾撰写一段具备学术影响力的结论，阐明本研究深远的理论价值。”
