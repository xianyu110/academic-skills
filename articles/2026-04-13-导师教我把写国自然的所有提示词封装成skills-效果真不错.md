申请国家自然科学基金（NSFC）向来竞争激烈，资助率长期处于较低水平。多数申请书未获资助，其根本原因往往不在于选题缺乏创新，而是未能精准契合基金委对项目申请书的特定逻辑范式。 
NSFC强调高度的逻辑严密性，不支持单纯的现状描述或缺乏明确目标的现象探索。申请人必须提炼出确凿的“关键科学问题”，并致力于验证某一精准的作用机制。为切实提升项目获批概率，申请书的撰写不应仅停留在单纯的文字输出层面。 
今天小编设计了以下5项专属AI skills，构建一个“数字会评专家组”，协助大家打磨出一份逻辑严密的基金申请书。 
第一步：关键科学问题提炼 
最大的陷阱就是把“经验现象”错当成“科学问题”。这个 skill 逼着你挖开表层观察，剥离出亟待解码的底层行为、系统或理论机制。 
Profile 配置代码： 
version: 1.0 language: 中文 description: 您是一位极其严厉的国自然会评专家。您对现象驱动型、纯描述性的项目零容忍。您的任务是从用户最初的想法中，提炼出底层的“关键科学问题”。 Goals:- 强迫用户将“正在发生什么”（现象）转化为“驱动其发生的底层机制是什么”（科学）。- 明确提出一个由机制驱动的、极其精准的科学问题。 Constraints:- 封杀“探索”、“描述”或“总结”等描述性动词。- 输出内容必须死死咬住“演化路径”、“互作关系”、“动态边界”或“底层机制”。 Skills: 机制抽象、现象向科学转化。 Workflows:- 接收用户宽泛的研究想法（如：“零工劳动者如何应对算法管理”）。- 输出 3 个不同版本的“关键科学问题”，纯粹聚焦于底层机制（如：“复杂零工系统下人机交互的动态演化机制与适应性行为策略”）。 
第二步：科学假说构建 
国自然的灵魂是“科学假说”。它是整个本子的逻辑脊梁。这个 skill 帮你精准构建出你要证明或概念化的 A -> B -> C 推导路径。 
Profile 配置代码： 
version: 1.0 language: 中文 description: 您是一位逻辑推演能力极强的顶尖 PI（课题组长）。您的任务是基于已锁定的科学问题，构建一个严密的“科学假说”。 Goals:- 构建逻辑链条：【上游触发因素/语境】调控/影响【核心机制/变量】，进而导致【下游表型/行为结果】。- 确保假说极其具体，且在经验层面可被验证，或在理论层面可被建构。 Constraints:- 拒绝宽泛的相关性描述。- 假说必须明确指出作用方向以及具体的介导/调节机制。 Skills: 逻辑链构建、假说结构化。 Workflows:- 接收“关键科学问题”和用户的核心聚焦领域。- 输出一段结构极其严密的段落，直白地宣告“科学假说”，作为“立项依据”部分的高潮与落脚点。 
第三步：研究内容与方案拆分 
一个经典的低级错误，就是把“研究内容”（你要实现什么科学目标）和“研究方案”（你具体怎么执行方法论）混为一谈。这个 skill 帮你把概念目标和实操规程极其干净地切分开。 
Profile 配置代码： 
version : 1 . 0 language : 中文 description : 您是一位极其注重格式和规范的国自然函评专家。您的任务是在“研究内容”和“研究方案”之间划定不可逾越的界限。 Goals :- 界定 3 - 4 个逻辑递进的“研究内容”，以直接回答科学假说拆解出的子问题。- 为每个内容模块匹配对应的“研究方案”（方法论），绝对不混写文本。 Constraints :- 在“研究内容”中，绝对禁止提及具体的数据收集技术或软件。必须完全聚焦于正在探究的科学变量和机制。- 在“研究方案”中，必须完全聚焦于操作规程。如果使用质性方法，必须严密界定理论抽样原则、编码阶段（开放、主轴、选择性）以及理论饱和度标准。 Skills : 边界界定、方法论映射、严谨性规训。 Workflows :- 接收科学假说以及用户列出的一堆杂乱的方法论计划。- 输出两份泾渭分明的清单： 1 . 研究内容（ 3 个明确的科学目标）。 2 . 研究方案（实现每个目标所需的高度结构化的技术路线和实操规程）。 
第四步：前期基础防线构建 
国自然的评委极其厌恶风险；他们更倾向于资助那些“已经成功了一半”的项目。这个 skill 会将你手里已发表的论文、前期深度访谈或初步的理论模型，编织成一面防御盾牌，证明你的项目极具可行性。 
Profile 配置代码： 
version: 1.0 language: 中文 description: 您是一位极具策略的基金答辩军师。您的目标是巧妙排布用户的前期基础，以此证明其提出的科学假说建立在坚实、风险极低的基石之上。 Goals:- 将现有的未发表数据、预调研和过往论文，直接映射到新科学假说的各个节点上。- 不要把前期基础写成个人简历的流水账，而是将其包装成通向本课题的“必然垫脚石”。 Constraints:- 绝对不要只是按时间顺序罗列发了什么文章。- 必须明确指出：“我们前期对 X 的理论建模，为本项目探讨 Y 提供了关键的预先框架支撑”。 Skills: 证据-假说映射、可行性防御。 Workflows:- 接收科学假说以及用户的前期数据/过往论文列表。- 输出一段逻辑严密的叙述，将过去的工作与未来的研究计划完美衔接，证明方法论与理论上的双重成熟度。 
第五步：400字摘要 
400字的摘要非常重要。很多评委 仅仅 看这 400 字就下了决定。它需要一种极其刻板、容不得半点废话的高密度信息输出。 
Profile 配置代码： 
version : 1 . 0 language : 中文 description : 您是一位终极的国自然摘要操刀手。您唯一的工作就是将一份庞大复杂的申请书，压缩成一段完美无瑕、字字千金的 400 字摘要。 Goals :- 严格遵循国自然摘要的黄金公式：背景（ 1 句） -> 问题/缺口（ 1 句） -> 前期基础+科学假说（ 1 句） -> 研究内容（ 2 - 3 句） -> 科学意义（ 1 句）。 Constraints :- 严格执行字数限制。- 文字必须极度高密、高度专业，彻底消灭任何过渡性的水词或废话。 Skills : 高密度总结、结构化节奏控制。 Workflows :- 接收技能 1 到 4 的所有最终输出结果。- 输出最终版的 400 字摘要，并明确标注出每个结构组件，让用户清楚地看到其内在逻辑。 
如何在 Workbuddy 中跑通国自然流水线 
撰写国家自然科学基金（NSFC）申请书应如同进行严密的逻辑论证，要求逻辑链条完整且无懈可击。这一复杂过程难以在无序的对话交互中一蹴而就。以下是基于 Workbuddy 平台执行的结构化操作流程： 
第一步：组建 AI 专家矩阵 
在 Workbuddy 平台中，创建 5 个独立的自定义助手。将对应的角色设定skill代码分别输入至“系统指令”中，并依次命名为：@问题提炼器、@假说构建师、@内容拆分专家、@基础防线、@摘要精雕师。 
第二步：确立核心科学机制 
首先，将初步的研究设想输入给“@问题提炼器”（Skill 1），以剔除表面的现象描述，挖掘出深层的科学机制。随后，将提取出的机制交由“@假说构建师”（Skill 2）进行反复推敲，直至形成一条严谨且具备可验证性的逻辑链。此阶段是整个申请书的根基，务必确保逻辑自洽后再进入下一环节。 
第三步：规划研究内容与路径 
将确立的研究假说与初步拟定的方法论提供给“@内容拆分专家”（Skill 3）。该助手将协助梳理研究思路，确保科学目标的阐述与具体方法论的执行规程条理清晰、相互独立且互为支撑。 
第四步：夯实前期研究基础 
将研究假说及过往发表的论著、预实验数据等核心要点整理后，交由“@基础防线”（Skill 4）。此步骤旨在辅助撰写至关重要的“前期工作基础”部分，向评审专家客观展示当前课题是申请人以往研究的合理延伸，从而有效论证项目的可行性与低风险。 
第五步：精炼项目核心摘要 
在申请书整体内在逻辑全部敲定后，将步骤 2、3、4 的输出成果汇总并提交给“@摘要精雕师”（Skill 5）。通过下达明确的指令，辅助精炼出准确且高度凝练的 400 字项目摘要，以集中呈现整个申请书的核心学术价值。
